# Data Privacy Reflection

## Reflection

**What steps can you take to ensure you handle data securely in your daily tasks?**

    In my daily tasks as a UX Design intern, I will treat user data with the same precision I use for design systems. Since Focus Bear handles "Special Category" data (ADHD/Autism status), I will ensure that any user feedback or behavior logs I review are stripped of Personally Identifiable Information (PII) before they enter the design phase. In design context, if I am testing a new habit-tracking interface, I will use nicknames or typical sample names rather than real customer names. This creates a safety buffer so that even if a prototype is accidentally shared, no real user privacy is compromised.

**How should you store, share, and dispose of sensitive information safely?**

    In safe-storing sensitive information, I will keep all project assets within the company’s encrypted cloud environment. I will avoid the convenience trap of saving sensitive spreadsheets or user interview recordings to my personal desktop or unencrypted USB drives. In sharing information, I will adopt a "Need-to-Know" sharing policy. Instead of using global "Anyone with the link" settings, I will invite specific teammates by their email addresses. Lastly, in disposing sensitive information, I will treat digital waste like physical confidential documents. Once a work cycle is over, I will audit my "Downloads" folder and permanently delete any local copies of user data or internal strategy briefs.

**What are some common mistakes that lead to data privacy issues, and how can they be avoided?**

    One common mistake in UX designing, especially when using Figma, it is easy to forget that a file is set to "Public" after a quick presentation. I will avoid this by defaulting all new files to "Restricted." Another common mistake is reusing same password for Focus Bear tools as my personal social media. I will avoid this by using a password manager and unique, complex passwords for every professional account.

## Task

**Identify at least one habit or practice you can adopt to improve data security in your role.**

    I will implement a weekly "Access Scrub" every Friday afternoon. This means I will spend 10 minutes reviewing the "Share" settings on every active Figma file I worked on that week. The goal remove any temporary guests, ensure no public links are active, and verify that "View Only" permissions are set for stakeholders who do not need to edit. This turns security into a routine rather than an afterthought.

**Document at least one key learning or security measure you will implement.**

    I will implement a "Privacy-First UI Prototype" rule. When building high-fidelity mockups that require habit data, I will use a library of generic, non-sensitive habits (e.g., "Drink 500ml water," "Morning Stretch"). This ensures that even if a screenshot of my design is used in a public-facing blog post or marketing materials, it contains zero "Special Category" data (like specific medication names or religious habits) that could potentially be traced back to the app's real-world users.
