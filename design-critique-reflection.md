# Design Critique Reflection

## Reflection

**What is the difference between personal opinion and objective UX critique?**

    A personal opinion is about what you like, such as saying "I think this font looks boring". An objective UX critique is about how it functions, "This font might be hard for a user with dyslexia to read quickly". Objective critique is always backed by design principles or user data. Personal opinions can be valid but should be framed in a way that focuses on the user experience rather than just personal taste.

**If someone gives vague feedback like "I don’t like this", how would you get more useful input?**

    I’d try to dig deeper by asking specific, open-ended questions. I might say: "That’s fair! Can you help me understand which part specifically feels off? Is it the layout, the colors, or does the flow feel a bit confusing for the user?" This turns a vague opinion into a useful design problem I can actually solve. It also shows that I value their input and want to understand their perspective better, which can lead to more constructive feedback in the future.

**How can you ensure that design critique sessions are productive rather than overwhelming?**

    To keep things from getting overwhelming, I should set the stage at the beginning. I’ll tell the team exactly what I’m looking for, like "I only need feedback on the checkout flow today, not the colors". Keeping the session focused on a specific goal prevents it from turning into a random venting session and keeps the feedback useful. I can also encourage everyone to be specific and constructive with their feedback, and remind them that the goal is to improve the user experience, not just to criticize. Finally, I can take notes during the session to make sure I capture all the useful feedback without getting lost in the noise.
