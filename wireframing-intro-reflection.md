# Wireframing Intro Reflection

## Reflection

**What are the benefits of starting with wireframes rather than jumping into high-fidelity design?**

    Starting with wireframes prevents you from liking a design too early. If you spend 5 hours making a beautiful high-fidelity screen and then find out the logic is broken, it’s painful to change. With wireframes, you stay flexible. It keeps the conversation focused on functionality rather than arguing about whether a button should be navy blue or royal blue. Wireframes are also faster to create and iterate on, allowing for more rapid feedback and adjustments. They help ensure that the core structure and user flow are solid before investing time in visual design.

**How can wireframes help identify usability issues early?**

    Since wireframes are so simple, you can see if a screen is too busy or if the path to finish a task is too long. For example, if I wireframe a "To-Do" screen and realize there are 15 different boxes, I can immediately see that it will cause cognitive overload for an ADHD user before I even start adding colors. Wireframes allow you to focus on the user experience and flow, making it easier to spot potential usability issues such as confusing navigation, unclear calls to action, or unnecessary steps in a process. By testing wireframes with users early on, you can gather feedback and make adjustments before investing time in high-fidelity design.

**If a stakeholder asks for a high-fidelity design too soon, how would you explain the value of wireframing?**

    I’d explain that wireframing is like the structural foundation of a house. You wouldn’t pick out the wallpaper before you’ve made sure the plumbing and electrical work. By starting with wireframes, we ensure the app is actually easy to use and functionally sound. This saves us from expensive re-dos later in the project when changes become much harder and costlier to implement. I’d also emphasize that wireframes allow us to focus on the user experience and flow without getting distracted by visual design elements. It’s a crucial step in the design process that helps us create a better product in the long run.
