# UX User Interviews Reflection

## Reflection

**If a user struggles with a feature during testing, how do you know if it’s a UX problem or a learning curve issue?**

    If a user struggles but figures it out quickly and never has the problem again, it might just be a learning curve. However, if multiple users get stuck at the same spot, or if the learning requires a manual or outside help, it’s a UX problem. At Focus Bear, the goal is "Zero Learning Curve", so the app should be so intuitive that it requires no explanation. If users consistently struggle with a feature, it indicates a UX problem that needs to be addressed.

**How can UX designers ensure they don’t influence user responses during an interview?**

    Stay neutral. Avoid nodding too much or saying "Correct" when they find something. Your body language should be a blank slate. If they ask, "Is this right?", turn it back on them by saying, "There’s no right or wrong answer here, I'm just interested in how you would naturally use it." This way, you encourage honest feedback without leading them to a specific answer.

**What’s the best way to document usability testing results so they are useful for design decisions?**

    To document design decisions effectively, use a scannable Findings Report that bridges the gap between raw data and actual Figma changes. Start with a clear goal and condense results into 3–5 Key Insights to keep the team focused on the biggest takeaways. Include direct quotes from users to add context and make the findings more relatable. Use visuals like screenshots or video clips to illustrate specific issues or successes. Finally, provide actionable recommendations for design changes based on the insights gathered, ensuring that the report is not just informative but also directly tied to the next steps in the design process.
